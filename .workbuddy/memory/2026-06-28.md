### 环形图绘制顺序修复 (18:13)
- **问题**：SVG `stroke-dashoffset` 正值会将图案**向后偏移**（逆时针），导致扇区排列顺序混乱
- **修复**：`stroke-dashoffset="${cumDash}"` → `stroke-dashoffset="${-cumDash}"`（负值向前推 = 顺时针）
- 原理：MDN — "A positive value shifts the dash pattern backward; a negative value shifts the dash pattern forward"
- 效果：最大扇区从 12 点方向开始，按降序顺时针依次排列

### 架构决策

| 维度 | 选型 | 理由 |
|------|------|------|
| 错误处理 | `Result<T>` sealed class + 7 种 `Failure` | 业务失败=值（编译期强制处理），bug=异常 |
| 数据流 | UI → Notifier → Service → Repository → DataSource | 单向，易测 |
| 状态管理 | Riverpod StateNotifier（按业务域拆分） | 4 个 Notifier 替代 1 个大 Provider |
| 持久化 | Hive 5 个 Box + Schema 版本号 | 与现有技术栈一致，零成本 |
| 启动 | AppBootstrap 编排 + SplashScreen + ErrorScreen | 启动失败可恢复 |

### 模块清单
- **core/** (7 个)：Result/Failure/常量/日志/格式化/启动/异常捕获
- **data/models/** (4 个)：HiveType 1-4，含 toEntity/fromEntity
- **data/repositories/** (4 个)：业务校验 + Entity↔Model 转换 + Result
- **data/sources/** (1 个)：HiveLocalDataSource 单例
- **domain/entities/** (5 个)：纯 Dart Entity，无框架依赖
- **domain/services/** (3 个)：StatsService/SearchService/BudgetService
- **providers/** (5 个)：按业务域拆分 + DI 总入口
- **services/** (2 个)：ExportService（CSV/JSON）+ BackupService（SHA-256 校验+回滚）

### 关键能力
1. **极速记账 3 步闭环**：从点击到保存 < 200ms
2. **统计聚合**：周/月/年 切换，4 种聚合（汇总/分类/日/月）
3. **搜索筛选**：关键词（备注/分类/金额）+ 时间 + 类型 + 排序 + 分页
4. **CSV/JSON 导出**：UTF-8 BOM 防乱码，share_plus 分享
5. **备份恢复**：SHA-256 校验 + 版本检查 + 恢复前自动回滚保险
6. **错误兜底**：全局异常捕获 + 启动校验 + 失败屏
