# 2026-06-29 工作日志

## 阶段九：APK 安装启动失败修复

### 问题
用户安装 `smart-ledger-v1.apk` 后打开失败：
```
LateInitializationError: Field '_txBox@580237182' has already been initialized.
```

### 根因
`HiveLocalDataSource.init()` 被并发调用两次：
1. `main.dart` 的 `initState()` → `_bootstrap()` → `AppBootstrap.run()` → `init()`
2. `providers.dart` 的 `initializationProvider` → `init()`

虽然 `_initialized` 守卫存在，但 `await` 让出控制权后两个调用都通过了守卫，都对 `late final` 变量赋值。

### 修复方案
1. **`main.dart`**：Hive 初始化提到 `runApp()` 之前，确保只执行一次
2. **`providers.dart`**：`initializationProvider` 去掉 `hive.init()` 调用
3. **`hive_local_data_source.dart`**：
   - `late final Box` → `Box?` 可空类型，避免重复初始化
   - 加 `_initializing` 标志，彻底防止并发
   - `Hive.isBoxOpen()` 判断是否需要重新打开（热重启安全）
4. **`app_bootstrap.dart`**：`run()` 里去掉 `init()` 调用，只做数据完整性校验

### 结果
- 重新构建 APK 成功：`smart-ledger-v1.apk`（44MB）
- 构建时间：21.4s（Gradle 缓存已热）
- 等待用户验证安装后是否能正常打开

### 技术经验
- Flutter `late final` 变量如果被多次赋值会抛 `LateInitializationError`
- 防并发不能只靠布尔标志，需要在 `await` 前后都检查
- Hive Box 打开操作应该放在 `runApp()` 之前，避免 Provider 初始化时并发触发
