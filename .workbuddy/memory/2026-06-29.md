## 阶段九：APK 安装启动失败修复（第二轮）

### 问题（持续）
用户安装 `smart-ledger-v1.apk` 后仍然打开失败，同样的 `LateInitializationError`。

### 根因（深层）
`initializationProvider` (FutureProvider) 在 `main()` 还在执行时就被 `ref.watch()` 触发，
导致 `HiveLocalDataSource.init()` 被并发调用两次。之前的 `_initializing` 标志不够可靠。

### 彻底修复方案
**完全移除异步初始化 Provider，把 Hive 初始化全部放到 `main()` 里同步完成**：

1. **`main.dart` 重写**：
   - `main()` 改为 `async`
   - Hive 初始化 + 数据校验全部在 `runApp()` 之前完成
   - 用 `SmartLedgerApp(initOk, initError)` 参数控制显示启动屏还是错误屏
   - 移除 `ConsumerStatefulWidget` / `initializationProvider` / 启动屏 Widget
   - 监听 `settingsNotifierProvider` 驱动主题切换

2. **`providers.dart` 清理**：
   - 删除 `initializationProvider`
   - 删除 `InitResult` 类

3. **`app_bootstrap.dart` 删除**：不再需要

4. **`category_notifier.dart`**：`load()` 里加 `_repo.seedDefaultsIfEmpty()`

### 结果
- 用户确认 **APK 可以正常打开** ✅

## 阶段十一：深色模式真正生效 + FAB 全 tab 显示

### 用户反馈（2026-06-29 上午）
1. **深色模式仍不行** —— 之前只在 main.dart 设置了 themeMode，但 widget 内部全部硬编码 `AppTheme.background/textPrimary/...`（浅色静态字段），所以即便 MaterialApp 切到 darkTheme，widget 依然用浅色
2. **+号按钮只在「记账」tab 显示** —— 之前是 `_tabIndex == 0 ? FAB : null`，用户希望全 tab 都可点 + 进入记账

### 根因分析
- AppTheme 既有 `lightTheme` / `darkTheme` 静态色板，但 widget 大量用 `AppTheme.background`（浅色）等 const 字段
- Theme.of(context) 切到 darkTheme 时，scaffold 背景变化，但 widget 内部文字/容器仍用 AppTheme 浅色常量
- 在深色模式下，浅色文字 (AppTheme.textPrimary) 在浅色背景下反而可能仍能看见，但和深色背景混在一起就视觉错乱

### 解决方案

**1. 新建 `core/utils/theme_utils.dart`**
- `ContextColors` 数据类封装背景/表面/文字/分割线/阴影等所有主题色
- `ContextColors.of(context)` 从 `Theme.of(context).colorScheme` 推断
- `context.colors` 扩展方便 widget 读取

**2. 全面替换 widget 内的硬编码 `AppTheme.*`**
- `lib/pages/home_page.dart`：HomeTab/StatsTab/RecordsTab/SettingsTab/CategoryMgmtPage/_StepNote/_topBar 全部改用 `context.colors`
- `lib/widgets/record_item.dart`、`category_grid.dart`、`numpad.dart`、`overview_card.dart` 全部同步

**3. ColorScheme 补全 `surfaceContainerHigh` / `outlineVariant`**
- lightTheme 与 darkTheme 都加上这两个字段，让 ContextColors 推断准确

**4. FAB 在所有 tab 都显示**
- `MainShell` 的 `floatingActionButton` 去掉 `_tabIndex == 0 ? FAB : null` 条件判断
- 永远显示一个 64×64 蓝色 + 号按钮，centerDocked 位置
- 在任何 tab 都能点 + 进入记账流程

### 构建结果
- **smart-ledger-v2.apk (51.8MB)** 已生成
- 提交：74b2942 → 推送到 GitHub

### 技术经验
- ⚠️ Material Design 3 ColorScheme 包含一组语义色 token（primary/onPrimary/surface/onSurface/error/surfaceContainerHigh/surfaceContainerHighest/outlineVariant 等），theme 切换应完全依赖 ColorScheme，让 widget 读取 `Theme.of(context).colorScheme.xxx` 而非 AppTheme 静态字段
- ⚠️ 旧 AppTheme 静态类在 M3 体系下是"过时"的反模式 —— 只能用于初始化 ThemeData 的入口值
- FAB 的 `_tabIndex == 0` 条件对"始终可记账"是反直觉设计，应保持常驻

## 阶段十三：Web 端 path_provider 报错修复

### 问题
用户在 Web 预览中看到 "MissingPluginException: No implementation found for method getApplicationDocumentsDirectory on channel plugins.flutter.io/path_provider"。根本原因：path_provider 在 Web 平台没有实现，但 Hive 初始化强制调用了它。

### 修复
在 `hive_local_data_source.dart` 的 `_initHive()` 加 web 平台分流：
```dart
if (kIsWeb) {
  await Hive.initFlutter();  // Web 用 IndexedDB，无需目录
} else {
  final dir = await getApplicationDocumentsDirectory();
  Hive.init(dir.path);
}
```

### 验证
重新构建后 18.2s 完成，HTTP 200 OK。

### Web 端限制清单（用户需要知道）
- ✅ Hive 数据存储（IndexedDB，刷新后保留）
- ✅ 深色模式切换
- ✅ FAB 全 tab 显示
- ✅ 记账流程、统计页、记录页、分类管理、设置页
- ❌ 导出 CSV/JSON（会报错，Web 端 dart:io 无文件系统）— 不影响主功能验证
