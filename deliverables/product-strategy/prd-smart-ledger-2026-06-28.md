# 智慧记账App 需求规格书（PRD）

> **产品名称**：智慧记账（Smart Ledger）
> **版本**：v1.0 MVP
> **撰写日期**：2026-06-28
> **撰写人**：析客（需求分析师）——产品战略团队
> **状态**：初稿待评审

---

## TL;DR

**一句话定位**：一把零成本、零打扰、零依赖的锋利记账小刀——为追求效率与数据自主的个人用户，打造3步完成录入的纯净记账体验。

**3个硬约束**：零成本实现 | 能装到手机上 | 核心功能闭环（收支记录+统计+分类管理）

**核心取舍**：砍掉一切需要账号、需要云服务、需要商业变现的功能——单人自用场景极致简化。

**推荐技术栈**：Flutter + Hive + Android APK = 全链路零成本，iOS需688元/年开发者费用。

**MVP目标周期**：4周可出可用的记账App。

## 一、产品目标

| # | 目标 | 衡量方式 |
|---|------|----------|
| G1 | **极速录入**——日常记账在3步内完成 | 新手首次记账≤3次点击完成；熟练用户≤2次点击 |
| G2 | **纯净体验**——零广告、零推送、零打扰 | App内商业相关内容数 = 0；主动推送通知数 = 0 |
| G3 | **数据自主**——数据100%本地可控，可随时带走 | 数据导出为CSV/JSON标准格式成功；迁移到新设备后数据完整可读 |

> 三目标正交：G1管交互效率、G2管体验纯净度、G3管数据独立性，互不冲突。

## 七、关键流程 / UI设计稿描述

### 7.1 核心记账流程（体验生死线——3步闭环）

```
┌─────────────────────────────────────────────────────────────┐
│                    记账录入3步闭环流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  打开App ──→ 主界面（底部大记账按钮居中）                      │
│     │                                                       │
│     ▼                                                       │
│  Step 1: 选择分类                                           │
│  ┌──────────────────────────────────┐                       │
│  │ 分类网格（6~8宫格快捷区）          │                       │
│  │ 常用分类优先排列                   │                       │
│  │ + 底部"更多分类"入口               │                       │
│  │ + 底部"新增分类"快捷入口           │                       │
│  └──────────────────────────────────┘                       │
│     │ 点击分类                                              │
│     ▼                                                       │
│  Step 2: 输入金额                                           │
│  ┌──────────────────────────────────┐                       │
│  │ 数字键盘（大按键，适配单手操作）    │                       │
│  │ 金额实时显示                       │                       │
│  │ 收入/支出切换开关（顶部）          │                       │
│  │ 备注快捷入口（小图标）             │                       │
│  └──────────────────────────────────┘                       │
│     │ 点击确认                                              │
│     ▼                                                       │
│  Step 3: 自动保存+反馈                                      │
│  ┌──────────────────────────────────┐                       │
│  │ 短暂toast"已记录 ¥XX"             │                       │
│  │ 自动返回主界面                     │                       │
│  │ 主界面余额实时更新                 │                       │
│  └──────────────────────────────────┘                       │
│                                                             │
│  ★ 熟练用户路径：打开App → 点分类 → 输金额 → 确认 = 2~3次点击 │
│  ★ 首次用户路径：打开App → 点分类(空列表) → 新增分类 → 回选 → │
│    输金额 → 硾认 = 4次点击（首次后降至2~3次）                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**交互原则**：
- 分类网格采用"常用优先"智能排序（使用频率最高的分类排最前）
- 数字键盘大按键设计，单手拇指可操作
- 收入/支出用顶部显眼切换开关，默认支出
- 保存后不跳转详情页，直接返回主界面——减少步骤

**UI设计原则**：
- 3Tab极简导航，不多不少
- 记账按钮是全App最重要的交互元素，视觉权重最高
- 列表默认按月分组，月内按日期倒序
- 所有操作不跳转新页面，尽量用弹窗/内联展开
- 无空状态焦虑——空列表即引导新建，不显示"暂无数据"干等

## 八、Non-goals（明确不做）

| # | 不做什么 | 原因 |
|---|---------|------|
| NG1 | **不做账号/注册体系** | 单人自用，无多端同步需求，账号是多余复杂度 |
| NG2 | **不做云端数据同步** | 数据自主底线 + 零成本约束，云服务有成本和依赖 |
| NG3 | **不做社交/分享功能** | 纯个人工具，无社交场景 |
| NG4 | **不做理财/投资推荐** | "纯净"是核心差异化，理财推送是竞品最大体验杀手 |
| NG5 | **不做广告/商业变现** | 个人工具无变现需求，广告是体验毒药 |
| NG6 | **不做多用户/家庭共享** | 单人闭环极致简化，多用户引入账号和权限复杂度 |
| NG7 | **不做iOS版本（MVP阶段）** | Apple Developer 688元/年是唯一硬支出，MVP只做Android |
| NG8 | **不做PWA/小程序** | 非真正的App体验，iOS PWA体验差 |
| NG9 | **不做预算强管控（如超预算锁定）** | 记账工具不是财务管控系统，预算是提示而非限制 |
| NG10 | **不做OCR/拍照自动识别（MVP）** | AI智能记账方向留到P2+，MVP先做好极速手动录入 |

## 十、待确认问题

| # | 问题 | 影响 | 建议默认立场 |
|---|------|------|-------------|
| Q1 | AI智能记账（语音/OCR）是否纳入P1？ | 如纳入则技术栈需引入本地推理引擎，工作量+4~6天 | MVP不纳入，留到P2+ |
| Q2 | 是否需要周视图统计？ | 影响统计页UI设计 | 默认只做月视图，周视图作为P1增强 |
| Q3 | 分类图标是否支持自定义上传？ | 影响分类管理复杂度 | MVP只提供内置图标库（50~80个），自定义上传为P2 |
| Q4 | 删除分类下的记录如何处理？ | 数据完整性影响 | 默认方案：删除分类时记录保留原分类名（冻结），不再可选 |
| Q5 | 金额是否需要小数点后2位强制？ | 用户体验一致性 | 默认支持最多2位小数，不强制 |
| Q6 | 首次打开App是否需要引导页？ | 新用户体验 | 不做引导页，空分类列表本身就是引导——直接引导新建分类 |
| Q7 | iOS版本何时启动？ | 成本支出时间 | MVP只做Android，Phase 2视需求决定是否投入688元做iOS |
| Q8 | 数据导出是否需要加密？ | 安全性 | 不加密——数据自主=用户自己掌控，加密反而是限制 |
